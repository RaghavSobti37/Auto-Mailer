🚀 Choosing an SMTP Provider

For bulk campaigns (10k+) you must use a transactional SMTP service. Gmail imposes a hard limit of ~500 emails/day.

ProviderSMTP HostPortFree TierIdeal For
Gmail smtp.gmail.com 587 Free Testing only (<500/day)
SendGrid smtp.sendgrid.net 587 100/day Startups & small lists
Amazon SES email-smtp.us-east-1.amazonaws.com 587 62k/month High-volume (20k–30k+)
Mailgun smtp.mailgun.org 587 1k/month Developers & APIs
Mailtrap (Testing) live.smtp.mailtrap.io 587 1k/month Safe sandbox testing

🔍 IMAP Configuration (Bounce Tracking)

To automatically detect bounced emails, AutoMailer needs to scan your inbox for delivery failure notifications. This requires IMAP to be enabled.

  1. Open Gmail in your browser.
  2. Click the Settings (cog) icon → See all settings.
  3. Go to the Forwarding and POP/IMAP tab.
  4. In the "IMAP access" section, select Enable IMAP.
  5. Click Save Changes at the bottom.

💡 Note: You must also use an App Password if you have 2-Step Verification enabled on your Google Account.

Google blocks standard password login for third-party apps. You must generate a dedicated 16-character App Password:

  1. Visit your Google Account Security Page →
  2. Enable 2-Step Verification (required).
  3. Search for "App passwords" in the security panel.
  4. Select app: Mail → Device: Other → Name it AutoMailer.
  5. Copy the 16-character code and paste it into the App Key field in the Studio.

⚠️ Keep this key private. AutoMailer stores it encrypted on your local machine only.

🖼️ Banner Embedding (Content-ID)

Standard email clients block external images by default. AutoMailer bypasses this by embedding your banner directly into the email source using the MIME Content-ID (cid:) standard.

  1. Click Select & Crop Banner Image in the Studio.
  2. Upload any JPG/PNG from your computer.
  3. Drag & resize the crop area (locked to 16:9 for consistent headers).
  4. Click Apply & Embed.
  5. AutoMailer automatically attaches the cropped image binary to every outgoing email — no CDN, no hosting.

📊 Conversion & Registration Tracking

AutoMailer provides full-funnel analytics without third-party trackers:

  • Open tracking: A 1×1 transparent pixel is silently embedded in every email. Loads = open recorded.
  • Click tracking: All links are routed through /track/click/…. A cookie (am_tracker) is dropped, then the user is redirected transparently.
  • Registration tracking: Place this pixel on your Thank You page:
    <img src="http://YOUR-HOST/track/register.gif" style="display:none;"> When it loads, AutoMailer reads the cookie and credits the conversion to the exact email that drove it.

All data is stored locally in data/tracking.db — a plain SQLite file you own entirely.

🏗️ Application Architecture

AutoMailer is a lightweight Python Flask application with zero build pipeline:

  • Backend: web_app.py — Flask routes, SMTP dispatch loop, tracking endpoints.
  • Database: tracking_db.py — SQLite via standard library. No ORM needed.
  • Frontend: Vanilla HTML/CSS/JS — no framework, no bundler. Runs in any browser.
  • Threading: Bulk sends run in a background threading.Thread so your browser never blocks.
  • Storage: Uploaded CSVs → data/uploads/. Tracking → data/tracking.db. Logs → logs/send_log.csv.

Start the server: py run_web.py → opens at http://127.0.0.1:5000
